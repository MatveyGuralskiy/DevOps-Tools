Introduction to Ansible for DevOps Beginners

About the Ansible

Ansible is an open-source automation tool that simplifies configuration management,
application deployment, and task automation.
It is particularly useful for DevOps professionals who aim to streamline
and automate various aspects of IT infrastructure management.

Getting Started

Installation Before using Ansible, ensure it is installed on your system.
You can install it using package managers like apt or yum. For example, on Ubuntu:

Open the Terminal (Ctrl+T)

First of all we need to update our packages:
sudo apt update

After that we can install our tool:
In Debian or Ubuntu:
sudo apt install ansible

In RedHat or CentOS:
sudo yum install ansible



Inventory

Ansible uses an inventory file to specify the hosts it will manage.
Create an inventory file (hosts.txt) and define your target hosts:

How to define your target hosts in inventory file:

[web_servers]
web1 ansible_host=192.168.1.101
web2 ansible_host=192.168.1.102
[name_of_group]
nameOfServer ansible_host=IP address

You can add for example staging group, production group and etc



Writing Playbooks

Playbooks are Ansible's configuration, deployment, and orchestration language.
They are written in YAML and describe the desired state of a system.

Example of Playbook:

Create yaml file for example playbook.yaml
At the start of the file we used 3 times "-"
---
And at the end of the file we used 3 times dot "."
...
Let's create a playbook


---
- name: Main purpose of Playbook
   hosts: Name of group from inventory file to work
   become: true  # Open all tasks with sudo permisions

   tasks:
   - name: Install Apache server
         apt: name=apache2 #or httpd state=latest

   - name: Restart Apache service
   service: name=httpd state=started enabled=yes
...



Run Playbook

Enter in terminal command:

ansible-playbook playbook.yml

This command will run the playbook (playbook.yml) on the hosts specified in the inventory file.

Ansible Modules
Ansible modules are reusable, standalone scripts that perform various tasks.
Explore the Ansible Module Index to discover modules for your specific needs.

Roles
Roles provide a framework for organizing playbooks and other files in a structured way.
They promote code reuse and make playbooks more modular.

Ansible Galaxy
Ansible Galaxy is a community hub for sharing Ansible roles.
You can find pre-written roles for common tasks, making it easier to leverage community knowledge.

Conclusion

Ansible simplifies automation and configuration management for DevOps professionals.
By understanding the basics of playbooks, modules, roles, and Ansible Galaxy,
you can efficiently manage and automate your IT infrastructure.
For more in-depth information, refer to the official Ansible documentation.
